pub enum ControlUI<Action = (), Err = ()> {
    Continue,
    Err(Err),
    NoChange,
    Change,
    Run(Action),
    Spawn(Action),
    Break,
}
Expand description

UI control flow.

This is the result type for an event-handler.

  • Continue - Continue with execution.
  • Err(Err) - Equivalent to Result::Err. Use the macro crate::tr to convert from Result.
  • Unchanged - Event processed, no UI update necessary.
  • Changed - Event processed, UI update necessary.
  • Action(Action) - Run some computation on the model.
  • Spawn(Action) - Spawn some computation on the worker thread(s).
  • Break - Break the event loop; end the program.

There are multiple continuation functions that work with these states. And the macros crate::tr! and crate::check_break!

impl<'a> HandleCrossterm<ControlUI<bool>> for Focus<'a>

source§

fn handle(&mut self, event: &Event, _: DefaultKeys) -> ControlUI<bool, ()>

Handle events. This is somewhat special as it doesn’t blend into the Action/Error types of the application, but returns its own action on focus change.

The idea is to react to the change, but not to cancel further processing of the event. This is crucial for handling mouse-events, otherwise the first click would focus, but do nothing otherwise. e.g. select a row in a table.
